    I recently started an React TS application and just realized that ESLint is not showing me the same errors/warnings within VSCode as it does on the command line. It seems to be basically running correctly as it does show me some errors:

    My config file is saved at project root within .eslintrc.js:

    /**
 * @type {import("eslint").Linter.Config}
 */
const config = {
  parser: '@typescript-eslint/parser',
  extends: [
    'plugin:react/recommended',
    'plugin:@typescript-eslint/recommended',
    'plugin:react/jsx-runtime',
    'plugin:prettier/recommended', // !!! Make sure this is always the last configuration in the extends array !!!
  ],
  parserOptions: {
    ecmaVersion: 2020,
    sourceType: 'module',
    ecmaFeatures: {
      jsx: true,
    },
  },
  rules: {
    'no-console': process.env.NODE_ENV === 'production' ? 'warn' : 'off',
    'no-debugger': process.env.NODE_ENV === 'production' ? 'warn' : 'off',
    '@typescript-eslint/no-unused-vars': ['warn', { argsIgnorePattern: '^_', varsIgnorePattern: '^_' }],
    '@typescript-eslint/ban-ts-comment': 'warn',
    'prettier/prettier': 'warn',
  },
  settings: {
    react: {
      version: 'detect',
    },
  },
};

module.exports = config;

    Neither the ESLint status icon/text, nor the debug output show me any obvious warnings or errors. Is there anything special I could look out for?

    Update

    I just realized that the error is shown by the plugin as well if I explicitly add '@typescript-eslint/explicit-module-boundary-types': 'warn', to the rules section in my config.

    My main questions is now:
    Why does ESLint follow different rules in the console and in the editor?
